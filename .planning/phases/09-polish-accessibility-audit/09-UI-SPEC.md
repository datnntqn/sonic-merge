## Haptics Contract

### Existing (already correct — do not change)

| Location | Trigger | Feedback |
|----------|---------|----------|
| `PillButtonStyle.makeBody` | `configuration.isPressed` | `.impact(weight: .light)` |
| `MergeSlotRow` | `isDragTouch` (drag start) | `.impact(weight: .medium)` |
| `MergeSlotRow` | `!isDragTouch` (drag end) | `.impact(weight: .light)` |
| `CleaningLabView` | `viewModel.hasDenoisedResult` | `.success` |
| `CleaningLabView` | `viewModel.errorMessage != nil` | `.error` |
| `CleaningLabView.abComparisonButton` | `viewModel.isHoldingOriginal` | `.impact(weight: .medium)` |
| `CleaningLabView.denoiseActionButton` | `viewModel.isProcessing` | `.success` |
| `AIOrbView` cancel button | `viewModel.isProcessing` | `.impact(weight: .medium)` |
| `LimeGreenSlider` | `isEditing` | `.selection` |

### Gaps to remediate (POL-01)

| Location | Current state | Required fix |
|----------|--------------|-------------|
| `MixingStationView` empty-state "Import Audio" button (line 122) | Raw `.clipShape(RoundedRectangle)`, no sensoryFeedback | Migrate to `PillButtonStyle(variant: .filled, size: .regular)` + inherits built-in haptic |
| `ExportFormatSheet` "Export Audio" button (line 60-71) | Raw `Button` with hardcoded background, no sensoryFeedback | Migrate to `PillButtonStyle(variant: .filled, size: .regular)` |
| `ExportProgressSheet` "Cancel Export" button (line 31) | `Button(role: .destructive)`, no sensoryFeedback | Add `.sensoryFeedback(.impact(weight: .medium), trigger: {boolean state})` OR migrate to `PillButtonStyle` with destructive tint; must retain `.destructive` role for accessibility |
| Toolbar buttons (`MixingStationView` toolbar: Import, Export, Denoise, Appearance) | SwiftUI system `ToolbarItem` buttons — system haptic via UIKit is typically absent | Add `.sensoryFeedback(.impact(weight: .light), trigger: {respective state bool})` at the toolbar level |

Buttons intentionally excluded from haptic requirement:
- Alert "Got It" button — system sheet, UIKit handles gesture; adding `sensoryFeedback` here is a no-op on system alerts.
- Context menu "Delete Clip" — system haptic provided by iOS long-press context menu presentation.
- List swipe-to-delete — system haptic provided by iOS swipe action.

## Accessibility Contract

### reduceTransparency (POL-03)

When `@Environment(\.accessibilityReduceTransparency)` is `true`, all glass/blur surfaces must render as solid opaque backgrounds.

| Surface | Current state | Required state when reduceTransparency=true |
|---------|--------------|---------------------------------------------|
| `SquircleCard(glassEnabled: true)` | `.ultraThinMaterial` + `surfaceGlass` tint | Solid `surfaceCard` fill — already implemented in Phase 6 |
| `LocalFirstTrustStrip` glassmorphism header | `.ultraThinMaterial` + `accentGlow@0.08` | Solid `surfaceCard` fill — already implemented in Phase 6 |
| `AIOrbView` outer bloom `Circle` | Radial gradient with `.blur(radius: 24)` | `.blur(radius: 8)` — already implemented in Phase 8 |
| `ExportFormatSheet` sheet background | Plain `VStack` with system background — no blur surface | No change needed; sheet uses system material (iOS-managed) |
| `ExportProgressSheet` sheet background | Plain `VStack` with system background — no blur surface | No change needed |

### reduceMotion (POL-03)

When `@Environment(\.accessibilityReduceMotion)` is `true`, all animated visuals must halt and serve a static equivalent.

| Animation | Current state | Required state when reduceMotion=true |
|-----------|--------------|---------------------------------------|
| AI Orb blobs (TimelineView + Canvas) | Paused via `shouldPause` when `reduceMotion` is true | Static t=0 render — already implemented in Phase 8 |
| PillButtonStyle scale press animation | Spring `.spring(response: 0.25)` | `nil` animation (scale stays 1.0) — already implemented in Phase 6 |
| MergeSlotRow drag scale/shadow | Spring `.spring(response: 0.28)` | `nil` animation — already implemented in Phase 7 |
| AI Orb progress ring `.easeOut(duration: 0.25)` | Animated trim | Needs audit: wrap in `if !reduceMotion { .easeOut } else { .linear(duration: 0) }` |
| `staleBanner` `.transition(.opacity)` in CleaningLabView | Fade in/out | Needs audit: wrap in `withAnimation(reduceMotion ? nil : .default)` at insertion site |

### Contrast (POL-03)

Minimum 4.5:1 contrast ratio required on all text. Targets to verify with a contrast tool:

| Text | Background (dark mode) | Background (light mode) | Status |
|------|----------------------|------------------------|--------|
| `textPrimary` (#F5F5F5 ~0.96 white) on `surfaceBase` (#000000) | ~20:1 | — | PASS |
| `textPrimary` (#1C1C1E) on `surfaceBase` (#FBFBFC) | — | ~19:1 | PASS |
| `textSecondary` (0.55 grey ~#8C8C8C) on `surfaceBase` (#000000) | ~4.6:1 | — | BORDERLINE — verify |
| `textSecondary` on `surfaceBase` (#FBFBFC light) | — | ~4.7:1 | BORDERLINE — verify |
| PillButton white label on Deep Indigo (#5856D6) | 3.2:1 | 3.2:1 | FAIL — existing known issue; `.ai` tint uses #1C1C1E on #A7C957 = 7.38:1 (PASS) |
| AIOrbView `orbLabelColor` (`accentAI` on `surfaceCard` #0F0F0F dark) | ~7.1:1 | — | PASS |
| ExportProgressSheet hardcoded `Color(red:0,0.478,1.0)` tint on white | — | — | Replace with semantic token; contrast inherits from accentAction |

Note on PillButton white label on Deep Indigo: contrast is 3.2:1 which is below 4.5:1. This is a pre-existing design decision made in Phase 6 (DS-03) under "Modern Spatial Utility" aesthetic. Phase 9 must document and escalate but must NOT change the established design token without explicit product decision. Executor must record this finding in the plan as a known accepted risk.
