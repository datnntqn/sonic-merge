Phase 4 typography rules:
- Toggle label "Normalize to -16 LUFS": `.font(.system(.body))` weight regular (400) — matches existing ExportFormatSheet body text.
- Toggle subtitle "Podcast standard (-16 LUFS)": `.font(.system(.caption))` weight regular (400), `.foregroundStyle(.secondary)` — matches existing secondary metadata pattern from ClipCardView.
- ExportProgressSheet dynamic title ("Exporting..." / "Exporting & Normalizing..."): `.font(.system(.headline))` weight semibold — no change from existing declaration (source: ExportProgressSheet.swift line 16).
- No new type ramp sizes introduced in Phase 4.

Weights in use (exactly 2):
- regular (400) — body, caption, secondary labels
- semibold (600) — headline, CTA button labels, sheet titles, percentage readout

|------|-----|-------------------|-------|

| Secondary (30%) | `#FFFFFF` | `Color.white` | Clip cards, sheet surfaces, intensity slider card background |

| Text Secondary | `.secondary` | System `.secondary` color | Captions, subtitles, duration labels |
| Destructive | `#FF3B30` | `.red` / `Button(role: .destructive)` | Cancel button in ExportProgressSheet only |
| Warning Surface | `#FFDF99` | `Color(red: 1.0, green: 0.88, blue: 0.6)` | Stale result banner background (CleaningLabView only) |

Accent (`#007AFF`) reserved for:
1. Export CTA button fill (ExportFormatSheet)
2. Import Audio CTA button fill (MixingStationView empty state)
3. ProgressView `.tint` (ExportProgressSheet)
4. Slider `.tint` (intensity slider in CleaningLabView)
5. Waveform bars in ClipCardView and WaveformCanvasView

7. Noise reduction intensity percentage text (CleaningLabView)
8. **Phase 4 addition**: LUFS Toggle `.tint` — uses accent blue to match all other interactive toggles/controls

AI accent (`#5856D6`) is present in UX-01 requirement but not yet used in Phase 3 CleaningLabView. Phase 4 does not introduce AI accent usage — LUFS normalization is a signal processing step, not an AI feature.

Destructive (`Button(role: .destructive)`) reserved for:
- "Cancel" button inside ExportProgressSheet only — no other destructive actions in Phase 4.

## Component Inventory

Components modified in Phase 4 (no new screens introduced):

### ExportFormatSheet (modified)

**Current state:** VStack with format Picker and Export button. Detent `.height(200)`.

**Phase 4 changes:**
- Add `@AppStorage("lufsNormalizationEnabled") var lufsEnabled: Bool = false` — default off (opt-in).
- Insert a `Toggle` row between the Picker and the Export button.
  - Toggle label: "Normalize to -16 LUFS" — `.font(.system(.body))`, `.foregroundStyle(Color(red: 0.110, green: 0.110, blue: 0.118))`
  - Toggle subtitle: "Podcast standard (-16 LUFS)" — `.font(.system(.caption))`, `.foregroundStyle(.secondary)`
  - Toggle `.tint(Color(red: 0, green: 0.478, blue: 1.0))`
  - Layout: `VStack(alignment: .leading, spacing: 4)` for label + subtitle, inside an `HStack` with the Toggle control.
  - Row horizontal padding: 24pt (matches Picker padding).
- Change signature: `onExport: (ExportFormat) -> Void` → `onExport: (ExportFormat, Bool) -> Void` (passes `lufsEnabled` flag).
- Expand sheet detent from `.height(200)` to `.height(280)` to accommodate the new row.

**Visual anatomy (top → bottom, with spacing):**
```
[20pt top padding]
"Export Format"   — .headline, centered
[24pt gap]
Segmented Picker (.m4a / .wav)   — .horizontal 24pt padding
[24pt gap]
HStack {
  VStack(leading) {
    "Normalize to -16 LUFS"   — .body
    "Podcast standard (-16 LUFS)"   — .caption, .secondary
  }
  Toggle (trailing)
}   — .horizontal 24pt padding
[24pt gap]
"Export" button   — full width, accent fill, 14pt vertical padding, cornerRadius 12
[32pt bottom padding]
```

### ExportProgressSheet (modified)

**Current state:** Hardcoded `Text("Exporting...")` title. Detent `.height(220)`.

**Phase 4 changes:**
- Add `isNormalizing: Bool` parameter (default `false` for backward compatibility).
- Title becomes: `isNormalizing ? "Exporting & Normalizing..." : "Exporting..."` — same `.headline` style, no layout change.
- No size, color, or layout changes beyond the dynamic string.
- Sheet height stays `.height(220)` — string length difference does not overflow at `.headline` size on any supported device width.

### MixingStationViewModel (export method signature change — UI contract only)

- `exportMerged(format:)` → `exportMerged(format:normalize:)` where `normalize: Bool`.
- ViewModel passes flag through to service layer. No new ViewModel state properties required.
- `isExporting`, `exportProgress`, `exportedFileURL`, `showShareSheet` — all existing state properties; no additions.
- After share sheet dismisses: `exportedFileURL = nil`, `exportProgress = 0`, sheet state cleared. (Source: CONTEXT.md — auto-present share sheet then reset to ready state.)

### Post-Export State Reset

Both MixingStationView and CleaningLabView:
- Share sheet dismissal triggers `dismissShareSheet()` / equivalent cleanup.
- No success toast, no "Share again" button — screen returns to its default ready state.
- This is existing behavior to be confirmed consistent; no new UI element is introduced.

## Interaction Contract

### Toggle Persistence
- `@AppStorage("lufsNormalizationEnabled")` key: `"lufsNormalizationEnabled"` (exact key — used by both ExportFormatSheet instances in MixingStationView and CleaningLabView since the sheet is a shared component).
- Default value: `false` — opt-in, not opt-out.
- Persists across app launches — user who enables it for podcasting does not need to re-enable each session.

### Sheet Height Behavior
- ExportFormatSheet expands from `.height(200)` to `.height(280)` to fit the toggle row.
- ExportProgressSheet remains `.height(220)` — dynamic title string fits without overflow.
- Both sheets remain `.interactiveDismissDisabled(false)` for ExportFormatSheet (user can dismiss before exporting) and `.interactiveDismissDisabled(true)` for ExportProgressSheet (export in progress — cannot dismiss).

### Export Flow Sequence (Phase 4 updated)

```
User taps Export toolbar button
  → ExportFormatSheet presents (.height(280))
    → User selects format + optionally enables LUFS toggle
      → User taps "Export"
        → ExportFormatSheet dismisses
          → ExportProgressSheet presents (.height(220))
            → Title: "Exporting & Normalizing..." if lufsEnabled, else "Exporting..."
              → Export + normalization completes
                → ExportProgressSheet dismisses
                  → Share sheet auto-presents (UIActivityViewController)
                    → User completes share action
                      → Share sheet dismisses
                        → State resets: exportedFileURL = nil, exportProgress = 0
                          → Screen returns to ready state (no toast, no confirmation)
```

### State Validation
- Export button in ExportFormatSheet: enabled when format is selected (always true — default is .m4a).
- LUFS toggle: always enabled in ExportFormatSheet — not gated on any pre-condition.
- Export toolbar button in MixingStationView: `.disabled(viewModel.clips.isEmpty || viewModel.isExporting)` — unchanged.
- Export toolbar button in CleaningLabView: `.disabled(!viewModel.hasDenoisedResult || viewModel.isProcessing)` — unchanged.
