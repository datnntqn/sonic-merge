# Phase 8 — UI Design Contract

> Visual and interaction contract for retrofitting the Cleaning Lab screen with the v1.1 "Modern Spatial Utility" design system AND introducing the AI Orb — a pulsating nebula sphere visualizer rendered via TimelineView + Canvas during denoising. Generated by gsd-ui-researcher. Pure view-layer phase — zero ViewModel or service changes.
>
> Platform: SwiftUI / iOS 17+ native. shadcn not applicable.
>
> **Upstream baseline:** `.planning/phases/06-design-system-foundation/06-UI-SPEC.md` (tokens + SquircleCard + PillButtonStyle + GlassmorphismHeader) and `.planning/phases/07-mixing-station-restyle/07-UI-SPEC.md` (PillButtonStyle Variant/Size matrix, spine-threading patterns, drag-animation conventions). Phase 8 **inherits all spacing, typography, color, and component primitives unchanged** and only defines net-new visual contracts specific to the Cleaning Lab + AI Orb.

### Phase 8 Accent Reservation — Lime Green `accentAI` (`#A7C957`)

Lime Green is the **Phase 8 visual signature**. It distinguishes AI work from the rest of the app, which uses Deep Indigo for non-AI primary actions. In Phase 8, Lime Green is reserved **exclusively** for:

1. **Progress ring around the AI Orb** — stroked arc from 0° to `progress * 360°`, Lime Green at 1.0 opacity.
2. **LimeGreenSlider track fill** — the filled portion of the slider (left of the thumb) is Lime Green at 1.0 opacity.
3. **Intensity percentage readout** (`"65%"`) — live number foreground uses `accentAI` (replaces current `accentAction` Deep Indigo).
4. **Primary CTA pill — `Denoise Audio` / `Re-process`** — `PillButtonStyle` variant is .filled with a Lime Green fill override (see [PillButtonStyle Lime Green override](#pillbuttonstyle-lime-green-override)). This is the ONLY place a PillButton renders in Lime Green instead of Deep Indigo.
5. **AI Orb rim glow** (outermost nebula layer only) — `accentAI` at 0.35 opacity blended via `.blendMode(.screen)` over the purple midlayer.
6. **A/B compare button "Denoised" label** — when the button reads "Denoised", the foreground text is `accentAI`. When it reads "Original" (hold state) the foreground inverts to `surfaceBase` on a filled `accentAction` background (unchanged from existing A/B convention).
7. **Stale banner "Re-process" button** — `PillButtonStyle(.filled, .compact)` with Lime Green fill override (same as primary CTA).

**Explicitly NOT used for:** body text, card backgrounds, the "Hold to compare" helper line, section headers, nav bar, waveform bars (those stay Deep Indigo to match Phase 7's `accentWaveform`), the A/B compare button's outer stroke (that stays `accentAction`).

### Deep Indigo `accentAction` / `accentGlow` continues to drive

- Waveform bars in `WaveformCanvasView` (unchanged — matches Phase 7)
- LimeGreenSlider **thumb fill** (Deep Indigo pill, 28pt diameter, Lime Green glow at 25% opacity radius 12 y:0 — thumb is the "control grab point", slider fill is the "value")
- A/B compare button stroke + filled "Original" state background
- Toolbar Export button and glassmorphism nav header tint
- SquircleCard drag-elevated shadow (none in Phase 8 — there's no drag in Cleaning Lab)
- The **core** of the AI Orb (innermost radial gradient stop — see below)

### AI Orb Color Story (locked)

The nebula sphere reads as a three-band radial composition. Color stops travel from Deep Indigo core → Purple midband → Lime Green rim glow, evoking "AI electricity churning inside a sphere".

| Layer (inside → outside) | Color | Token | Opacity | Blend mode |
|---|---|---|---|---|
| Core (0% → 25% radius) | Deep Indigo `#5856D6` | `accentAction` | 0.95 | `.normal` |
| Mid 1 (25% → 55% radius) | System Purple `#AF52DE` | `accentGradientEnd` | 0.75 | `.screen` |
| Mid 2 (45% → 75% radius) | System Purple `#AF52DE` | `accentGradientEnd` | 0.50 | `.screen` |
| Rim (70% → 100% radius) | Lime Green `#A7C957` | `accentAI` | 0.35 | `.screen` |
| Outer glow bloom (100% → 120% radius, outside canvas) | Lime Green `#A7C957` | `accentAI` | 0.18 | `.screen` (rendered via `.blur(radius: 24)` on a separate layer) |

Rationale: Deep Indigo as the core anchors the orb to the brand; Purple is the mid-density nebula tone (reusing Phase 7's `accentGradientEnd`); Lime Green at the rim is the unmistakable "AI is active" signature per CL-02. Using `.blendMode(.screen)` lets the layers additively brighten rather than mudding into gray.

### Dark mode — WCAG contrast verified

Lime Green `#A7C957` contrast table at 1.0 opacity:

| Background | Contrast | WCAG rating | Usage |
|---|---|---|---|
| `surfaceBase` light `#FBFBFC` | **1.72:1** | ⚠️ fails text contrast | OK for **graphics** (progress ring, slider track, orb rim) — HIG exempts non-text decorative graphics from 4.5:1. NOT used for text on light surfaceBase. |
| `surfaceCard` light `#FFFFFF` | **1.71:1** | ⚠️ fails text contrast | Same — OK for graphics, NOT for text. |
| `surfaceBase` dark `#000000` | **12.04:1** | ✅ AAA | OK for text and graphics. |
| `surfaceCard` dark `#0F0F0F` | **11.32:1** | ✅ AAA | OK for text and graphics. |

**Consequence — critical contract for both researcher AND checker:** Lime Green is **ONLY** used for **graphics** (progress ring, track fill, button fill, orb rim) and for **text in dark mode**. The intensity percentage readout "65%" on a `surfaceCard` background:

- **Dark mode:** `accentAI` directly — passes AAA at 11.32:1 ✓
- **Light mode:** `accentAI` still used, BUT the readout sits inside the `intensitySlider` SquircleCard which has a white (`surfaceCard`) background where Lime Green at `.title3.semibold` is **1.71:1 — failing**. Mitigation: in light mode, the percentage text uses Deep Indigo `accentAction` (6.24:1 — passes AA large) instead of Lime Green. This is the ONE mode-conditional color swap in Phase 8.

Implementation: `colorScheme == .dark ? semantic.accentAI : semantic.accentAction` on the percentage `Text` foreground only.

Similarly for the **"Denoising…"** label text below the orb: `accentAI` in dark mode, `accentAction` in light mode.

**All graphics (progress ring, slider track, button fills, orb rim) use Lime Green in both modes — they are decorative, not text, and HIG exempts them from 4.5:1.**

### PillButtonStyle Lime Green Override

Phase 8 needs the primary CTA pill to render in Lime Green instead of Deep Indigo. The Phase 7 `PillButtonStyle.Variant` enum does not yet support this. Rather than adding a new enum case, Phase 8 introduces a **tint parameter**:

```swift
struct PillButtonStyle: ButtonStyle {
    enum Variant { case filled, outline }
    enum Size { case regular, compact, icon }
    enum Tint { case accent, ai }  // NEW (Phase 8)

    let variant: Variant
    let size: Size
    let tint: Tint

    init(variant: Variant = .filled, size: Size = .regular, tint: Tint = .accent) {
        self.variant = variant
        self.size = size
        self.tint = tint
    }
    // ... backgroundFill branches on (variant, tint):
    //     (.filled, .accent) → accentAction (Deep Indigo)   — Phase 6/7 default
    //     (.filled, .ai)     → accentAI     (Lime Green)    — Phase 8 primary CTA
    //     (.outline, .accent) → clear + accentAction stroke — Phase 7 gap row
    //     (.outline, .ai)     → clear + accentAI stroke     — Phase 8 A/B compare unselected
}
```

**Default `.accent` preserves Phase 6/7 behavior byte-for-byte.** No existing call site changes unless it opts into `.ai`.

**Contrast of white label on Lime Green fill:** white (`#FFFFFF`) on `#A7C957` = **1.98:1 — fails**. Lime Green pills therefore use a **dark label** — specifically `textPrimary` (`#1C1C1E` light / `#F5F5F5` dark).

- `#1C1C1E` on `#A7C957` = **7.38:1** ✅ AAA
- `#F5F5F5` on `#A7C957` = (light-mode dark-text swap) — not applicable, Lime Green pills use the dark `#1C1C1E` label in BOTH modes for brand consistency. In dark mode this gives `#1C1C1E` on `#A7C957` = 7.38:1 still ✅ AAA.

Implementation inside `PillButtonStyle.labelColor`:
```swift
private var labelColor: Color {
    switch (variant, tint) {
    case (.filled, .accent): return .white                              // Phase 6/7 default
    case (.filled, .ai):     return Color(uiColor: SonicMergeTheme.ColorPalette.primaryText)  // #1C1C1E
    case (.outline, _):      return Color(uiColor: semantic.textPrimary)
    }
}
```

The inner glow overlay (white linear gradient at 25% opacity) is **retained** for Lime Green filled pills — on `#A7C957` it reads as a soft highlight at the top edge, maintaining the "pressed glass" quality from Phase 6.

## Component Contracts

### AI Orb (`AIOrbView`) — the defining visual of Phase 8

A 240pt × 240pt pulsating nebula sphere rendered via `TimelineView(.animation)` + `Canvas`. Three layered `RadialGradient` blobs modulated by sine-wave phase offsets produce the illusion of churning plasma. Visible only while `viewModel.isProcessing == true`; otherwise the hero container renders a **static composed still** of the same color field (the reduceMotion fallback doubles as the idle-state graphic).

| Property | Value |
|----------|-------|
| File | `SonicMerge/Features/Denoising/AIOrbView.swift` (NEW) |
| Canvas frame | **240pt × 240pt**, fixed, square |
| Outer bloom frame | 288pt × 288pt (24pt of blurred overflow on each side) — achieved via `.blur(radius: 24)` on a separate outer layer, clipped to 288pt |
| Placement | Inside `CleaningLabView`'s hero `SquircleCard`, centered horizontally, `Spacing.xl` (32pt) top + bottom padding |
| Implementation primitive | `TimelineView(.animation(minimumInterval: 1.0 / 60.0, paused: reduceMotion)) { timeline in Canvas { ctx, size in … } }` |
| FPS target | **60 FPS** on A14+; `Canvas` auto-throttles on older devices |
| Blob count | **4 layered radial blobs** per frame (core + 3 nebula bands). Keeps GPU cost below the 2-blur budget inherited from STATE.md. |
| Blob radius modulation | Each blob's radius = `baseRadius * (1 + 0.08 * sin(t * frequency + phaseOffset))` — i.e. ±8% radius pulse |
| Blob position modulation | Each blob's center drifts in a small Lissajous figure: `center + (cos(t * fx + px) * 12, sin(t * fy + py) * 12)` — 12pt max drift, sub-threshold for reduceMotion concern in isolation but suppressed by the reduceMotion branch anyway |
| Per-blob phase offsets (radians) | `[0.0, 1.57, 3.14, 4.71]` (0°, 90°, 180°, 270°) |
| Per-blob frequency multipliers (Hz) | `[0.35, 0.50, 0.65, 0.80]` — prime-ish ratios so the blobs never resync into a visible rhythm |
| Core blob | Center `(120, 120)`, baseRadius 40pt, RadialGradient `[accentAction@0.95, accentAction@0]`, blendMode `.normal` |
| Mid blob 1 | Center drifts, baseRadius 70pt, RadialGradient `[accentGradientEnd@0.75 at center, accentGradientEnd@0 at edge]`, blendMode `.screen` |
| Mid blob 2 | Center drifts, baseRadius 95pt, RadialGradient `[accentGradientEnd@0.50, accentGradientEnd@0]`, blendMode `.screen` |
| Rim blob | Center `(120, 120)` (anchored), baseRadius 115pt, RadialGradient `[accentAI@0, accentAI@0.35 at 0.7, accentAI@0]`, blendMode `.screen` — donut ring at the outer edge |
| Outer bloom | Full 288pt square, solid `accentAI@0.18`, masked by a radial gradient fading to transparent at 50% radius, rendered on a separate `View` layer with `.blur(radius: 24).blendMode(.screen)` (outside `Canvas` — cheaper this way) |
| Saturation boost | `.saturation(1.15)` on the whole orb stack for extra punch — subtle, well within HIG |
| Sphere clip | The entire `Canvas` is NOT circle-clipped — the rim blob's gradient falloff naturally creates the sphere silhouette. This avoids a hard edge and preserves the nebula bloom. |
| Animation math input `t` | `timeline.date.timeIntervalSinceReferenceDate` — an always-advancing double. Use `t` directly in sin/cos (frequency expressed as radians/sec × 2π). |
| Progress ring overlay | `Circle().trim(from: 0, to: viewModel.progress).stroke(accentAI, style: StrokeStyle(lineWidth: 4, lineCap: .round)).rotationEffect(.degrees(-90)).frame(width: 256, height: 256)` — 8pt outside the 240pt orb, starts at 12 o'clock, sweeps clockwise. Ring is inside `AIOrbView` so it's colocated with the orb and participates in the same container padding. |
| Progress ring animation | `.animation(.easeOut(duration: 0.25), value: viewModel.progress)` — smooth each discrete progress update from the ViewModel |
| Label below orb | `Text("Denoising…").font(.title3).fontWeight(.semibold).foregroundStyle(colorScheme == .dark ? semantic.accentAI : semantic.accentAction)` positioned 12pt below the ring. **Ellipsis is a literal one-char ellipsis `…`, NOT three periods.** |
| Percent readout below label | `Text("\(Int(viewModel.progress * 100))%").font(.subheadline).foregroundStyle(semantic.textSecondary).monospacedDigit()` |
| Accessibility label | `.accessibilityLabel("AI denoising in progress, \(Int(viewModel.progress * 100)) percent complete")` — dynamic, updates on each progress tick |
| Accessibility traits | `.accessibilityAddTraits(.updatesFrequently)` — VoiceOver announces changes without interrupting |
| `.accessibilityHidden` | false — orb IS an essential status indicator. VoiceOver announces it. |

#### reduceMotion fallback (static composed still)

When `@Environment(\.accessibilityReduceMotion) == true`:

1. The `TimelineView` is constructed with `.paused: true` — the `Canvas` draws **exactly one frame** at `t = 0` and never re-invokes.
2. At `t = 0`, all phase offsets are at their starting positions — blob centers are at their Lissajous origin, radii are at base (no sin modulation, since `sin(0) = 0`).
3. The progress ring **still animates** — `progress` updates are not motion (they convey information). `.animation(.easeOut(duration: 0.25), value: progress)` is retained.
4. The outer bloom `.blur(radius: 24)` is retained (blur is not motion).
5. The "Denoising…" label is retained.

The resulting static composition is a **deliberately composed** symmetric nebula — Deep Indigo core, Purple midband, Lime Green rim — that reads identically whether the user just arrived (no motion) or is watching the pulse (full motion). Not a paused mid-animation frame; the math is designed so `t=0` IS the canonical "at rest" composition.

#### Idle state (no denoising in progress, no result yet)

When `viewModel.isProcessing == false && viewModel.hasDenoisedResult == false`:

- AI Orb still renders the **same static composition** (reduceMotion branch is reused — `paused: true` regardless of the reduceMotion environment value when `!isProcessing`).
- Progress ring is **hidden** (`if viewModel.isProcessing` wrapping the ring).
- Label switches to "Ready to denoise" (`.title3.semibold`, `textSecondary` foreground — muted because it's passive).
- Percent readout hidden.

#### Success state (denoising finished, result available)

When `viewModel.isProcessing == false && viewModel.hasDenoisedResult == true`:

- Orb stays static (paused).
- Progress ring shows full 360° (`trim: 0…1.0`) in Lime Green — remains visible as a "job done" indicator.
- Label switches to "Denoised" (`.title3.semibold`, `accentAI` in dark / `accentAction` in light).
- Percent readout hidden.

#### GPU budget

- 1 blur layer (`.blur(radius: 24)` on outer bloom)
- 1 saturation filter (`.saturation(1.15)`)
- 1 glassmorphism nav header (inherited from Phase 6)

**Total: 2 blur-equivalent layers** — at the v1.1 GPU budget ceiling. The `Canvas` itself does not count toward the blur budget. No additional blurs permitted in Cleaning Lab. If the planner needs another visual that requires blur, it's a BLOCKER.

Built from primitive shapes — NOT the `Slider` control — because the iOS Slider's `.tint(_:)` only colorizes the filled track and leaves the thumb at system default. | Track height | **6pt** |
| Track corner radius | 3pt (pill — half of height) |

| Thumb shape | `Circle()` |
| Thumb diameter | **28pt** |
| Thumb fill | `Color(uiColor: semantic.accentAction)` (Deep Indigo `#5856D6`) |
| Thumb stroke | 1pt white at 0.3 opacity (subtle bevel for legibility on dark mode) |
| Thumb glow | `.shadow(color: accentAI.opacity(0.35), radius: 12, x: 0, y: 0)` — Lime Green halo around Deep Indigo thumb (matches the "control grabs the value color" metaphor) |

## Implementation Constraints (Hard)

These are NOT design decisions — they are guardrails the planner/executor must respect. A violation blocks the phase.

1. **No ViewModel / service changes.** `CleaningLabViewModel`, `NoiseReductionService`, `AudioMergerService`, `WaveformService` are frozen. If a visual decision would require a new `@Published` / `@Observable` property, the decision is wrong. The progress modal removal uses the EXISTING `viewModel.isProcessing` / `viewModel.progress` / `viewModel.cancelDenoising()` surface.
2. **iOS 17 minimum.** No `MeshGradient`, no `Spline`, no iOS 18+ APIs. All animation is `TimelineView(.animation)` (iOS 15+), `Canvas` (iOS 15+), `LinearGradient` / `RadialGradient` (iOS 13+), and `.sensoryFeedback` (iOS 17+). Build must succeed on iOS 17 simulator.
3. **Max 2 blur layers per screen** (v1.1 GPU budget, inherited from STATE.md). Cleaning Lab's blur count: 1 layer (glassmorphism nav header — inherited Phase 6) + 1 layer (AI Orb outer bloom `.blur(radius: 24)`) = **2 layers — at ceiling**. Phase 8 may not add any more blurs. The LimeGreenSlider thumb glow uses `.shadow(...)` not `.blur(...)` — doesn't count.
4. **No new token values.** `accentAI` (#A7C957), `accentGradientEnd` (#AF52DE), and all Phase 6/7 tokens keep their exact values. Phase 8 introduces **zero** new color tokens. The `PillButtonStyle.Tint` enum is a new API surface, not a new token.
5. **No hardcoded colors.** Zero `Color(red:green:blue:)` literals, zero `Color(hex:)` helpers, zero `.white`/`.black` (except as return values from semantic computed properties). Every color MUST resolve via `semantic.*` or `SonicMergeTheme.ColorPalette.*`.
