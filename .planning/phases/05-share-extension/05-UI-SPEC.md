## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, inline padding |
| sm | 8px | VStack item spacing, compact rows |
| md | 12px | Card horizontal padding, HStack spacing (used in ClipCardView, staleBanner) |
| lg | 16px | Screen horizontal padding, card vertical padding (used in CleaningLabView .padding(.horizontal, 16)) |
| xl | 20px | Section VStack spacing (used in CleaningLabView .padding(.vertical, 20)) |
| 2xl | 28px | Sheet top padding (used in ExportProgressSheet .padding(.top, 28)) |
| 3xl | 32px | Sheet horizontal ProgressView padding, sheet bottom padding |

Exceptions:
- Touch targets: minimum 44pt height for all tappable controls (iOS HIG). Waveform thumbnail frame is exactly 44pt tall (ClipCardView `.frame(width: 60, height: 44)`).
- HUD card: fixed height 220pt (matches ExportProgressSheet `.presentationDetents([.height(220)])`).
- Corner radius: 12pt for all cards and buttons (established pattern across ClipCardView, CleaningLabView, ExportProgressSheet). Exception: waveform thumbnail uses 6pt corner radius.

**Exception justification: 12px, 20px, and 28px are existing production constants extracted from Phase 2–4 Swift sources (ExportProgressSheet.swift, CleaningLabView.swift, ClipCardView.swift). They cannot be rounded to the standard set without breaking visual parity with the main app.**

Source: Extracted from ClipCardView.swift, CleaningLabView.swift, ExportProgressSheet.swift, MixingStationView.swift.

## Typography

All fonts use `.system()` API — no hardcoded font name.

| Role | Size | Weight | Line Height | SwiftUI API |
|------|------|--------|-------------|-------------|
| Heading | `.headline` (~17pt) | semibold (600) | 1.2 | `.font(.system(.headline))` |
| Body | `.body` (~17pt) | regular (400) | 1.5 | `.font(.system(.body))` |
| Label | `.subheadline` (~15pt) | semibold (600) | 1.3 | `.font(.system(.subheadline, weight: .semibold))` |
| Caption | `.caption` (~12pt) | regular (400) | 1.3 | `.font(.system(.caption))` |

**HUD-specific typography:**
- Filename text: `.subheadline` weight `.semibold` — single line, truncated with `.lineLimit(1)` and `.truncationMode(.middle)`
- Status text ("Adding to SonicMerge..."): `.body` weight regular
- Completion text: `.body` weight `.semibold`

Source: Established across ExportProgressSheet.swift, ClipCardView.swift, CleaningLabView.swift. Four sizes maximum; no additional sizes may be introduced in this phase. Two weights only: regular (400) and semibold (600).

## Component Inventory

The Share Extension UI consists of a single screen: the auto-dismiss HUD sheet. All components must mirror the main app aesthetic.

### HUD Sheet (`ShareExtensionViewController`)

A `UIViewController` subclass hosting a SwiftUI view via `UIHostingController`. Presented as the extension's principal class.

**Layout (top to bottom, centered):**

```
┌─────────────────────────────────┐  ← white card, cornerRadius 12pt
│                                 │
│   [SF Symbol: doc.badge.plus]   │  ← 32pt, accent #007AFF
│                                 │
│   "Adding to SonicMerge..."     │  ← .body, regular, #1C1C1E
│   [filename truncated middle]   │  ← .subheadline, semibold, .secondary
│                                 │
│   ────────[ProgressView]─────── │  ← linear, tint #007AFF, indeterminate
│                                 │
└─────────────────────────────────┘
```

- Card: `Color.white`, `cornerRadius(12)`, shadow `Color.black.opacity(0.04)` radius 4 x:0 y:2
- Sheet height: 220pt fixed (`presentationDetents([.height(220)])`)
- Horizontal padding inside card: 32pt (matches ExportProgressSheet)
- Vertical stack spacing: 20pt (matches ExportProgressSheet)
- Top padding: 28pt, bottom padding: 32pt (matches ExportProgressSheet)
- `interactiveDismissDisabled(true)` — user cannot swipe to dismiss during copy

**States:**

| State | Visual |
|-------|--------|
| Copying | Indeterminate `ProgressView` spinning, filename shown |
| Success | Brief: text changes to "Added!" then auto-dismisses after 0.3s |
| Error | Text changes to "Could not add file", `dismissButton` appears (destructive role) |

**Success auto-dismiss:** 0.3s delay after completion text change, then `extensionContext?.completeRequest(returningItems: nil)`.

**Error dismiss:** Single "Dismiss" button, `Button(role: .destructive)`, triggers `extensionContext?.cancelRequest(withError:)`.

### No additional components

This phase introduces only `ShareExtensionViewController` + its hosting SwiftUI HUD view. No changes to existing screens.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| User taps "SonicMerge" in iOS Share Sheet | System presents `ShareExtensionViewController` as a sheet |
| HUD appears | Immediately shows indeterminate spinner + filename + "Adding to SonicMerge..." |
| File copy completes | Text transitions to "Added!" — no user action needed |
| Auto-dismiss | 300ms after "Added!" text appears, extension calls `completeRequest` and dismisses |
| `NSItemProvider` load fails | Text changes to "Could not add file" + "Dismiss" button |
| User taps "Dismiss" on error | Extension calls `cancelRequest(withError:)` and dismisses |
| Main app opens via URL scheme | Main app receives `sonicmerge://import?file=<filename>`, calls `importFiles([url])` |

**No confirmation step.** Decision D-01 (CONTEXT.md): auto-dismiss only — no user confirmation required.

**Memory constraint.** Use `NSItemProvider.loadFileRepresentation` (streams to temp file), never `loadDataRepresentation`. This is a hard constraint from CONTEXT.md decision on 30 MB+ files (SC-2).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Sheet title / HUD status — loading | "Adding to SonicMerge..." |
| HUD filename line | `{filename}` — truncated middle if longer than one line |
| HUD status — success | "Added!" |
| HUD status — error | "Could not add file" |
| Error dismiss button | "Dismiss" |
| Main app — no action on arrival | (none — `onOpenURL` handler is silent; MixingStation already shows clips) |

**No empty state copy needed** for the extension itself — the extension only appears when a file is being shared (iOS activates it only when the share payload matches the `NSExtensionActivationRule`).

**No destructive confirmation copy** — this phase has no destructive actions. "Dismiss" on error is neutral, not destructive, but uses `.destructive` button role for visual de-emphasis per established app pattern (ExportProgressSheet cancel button pattern).
