    Actions:
    - startDenoising(mergedFileURL: URL) — triggers merge→denoise pipeline
    - cancelDenoising() — cancels in-flight Task, discards result, resets isProcessing
    - onIntensityChanged(_ value: Float) — calls noiseReductionService.blend() on actor; writes blended buffer to temp file; reloads denoisedPlayer
    - holdBegan() — if hasDenoisedResult: switch to originalPlayer at currentTime
    - holdEnded() — switch back to denoisedPlayer; UIImpactFeedbackGenerator(.medium).impactOccurred()
    - markClipsChanged() — if hasDenoisedResult: showsStaleResultBanner = true

    Pipeline sequence in startDenoising():
    1. Set isProcessing = true, progress = 0.0
    2. Create temp URL in FileManager.default.temporaryDirectory for denoised .wav
    3. Call noiseReductionService.denoise(inputURL: mergedFileURL, outputURL: tempURL)
    4. Consume AsyncStream: for await p in stream { progress = p }
    5. On stream finish: call waveformService.generate() for denoised file
    6. Set waveformPeaks, hasDenoisedResult = true, isProcessing = false
    7. Prepare both AVAudioPlayers with prepareToPlay()
    8. Start denoisedPlayer.play() (autoplay per CONTEXT.md)
    9. On error: set errorMessage, isProcessing = false
    10. On cancel: task.cancel() → break stream loop → discard temp file

- Never call AVAudioPlayer or NoiseReductionService directly from synchronous context on @MainActor without Task{}
- Actor hop for blend: `let blended = await noiseReductionService.blend(original: originalFrames, denoised: denoisedFrames, intensity: value)`
- Write blended buffer to new temp .wav and reinitialize denoisedPlayer so AVAudioPlayer plays the mixed version
- Both players must call prepareToPlay() before holdBegan()/holdEnded() is ever called
- UIImpactFeedbackGenerator must be called on holdEnded() release (UX-02)
- import UIKit for UIImpactFeedbackGenerator
- import AVFoundation, Accelerate
