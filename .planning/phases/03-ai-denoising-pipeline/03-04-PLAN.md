**CleaningLabView.swift**

Create at SonicMerge/Features/Denoising/CleaningLabView.swift.

The view receives `mergedFileURL: URL` and a `MixingStationViewModel` reference (or the sorted clips) from the NavigationLink call site. It owns its CleaningLabViewModel as a @State property.

Layout (vertical stack, NavigationTitle "Cleaning Lab"):

1. **Stale result banner** (conditional — shown when viewModel.showsStaleResultBanner):
   - Yellow/orange background banner at top: "Clips have changed. Re-process to update the denoised audio."
   - Button: "Re-process" → calls viewModel.startDenoising(mergedFileURL:)
   - Display only when viewModel.showsStaleResultBanner == true AND viewModel.hasDenoisedResult == true

2. **Waveform display** (shown when viewModel.hasDenoisedResult):
   - Full-width Canvas/GeometryReader waveform using viewModel.waveformPeaks (50 peaks)
   - Vertical bars, symmetrical top+bottom like ClipCardView but full width
   - A playback position indicator line (scrub line) — read from AVAudioPlayer via a timer or ViewModel property
   - Show placeholder text "Processing..." when isProcessing=true and peaks are empty

3. **Intensity slider**:
   - Label: "Noise Reduction" with value badge showing percentage (e.g., "75%")
   - `Slider(value: Binding(get: { viewModel.intensity }, set: { viewModel.onIntensityChanged($0) }), in: 0...1)`
   - Disabled while viewModel.isProcessing == true
   - Show only when viewModel.hasDenoisedResult == true (or always visible, dimmed before processing)

4. **A/B comparison button** (shown when viewModel.hasDenoisedResult):
   - SwiftUI button with long-press gesture: `.onLongPressGesture(minimumDuration: 0, pressing: { isPressing in ... })`
   - When pressing=true: viewModel.holdBegan()
   - When pressing=false: viewModel.holdEnded()
   - Label: isHoldingOriginal ? "Original" : "Denoised" with visual state change (filled vs outlined)
   - Below button: small helper text "Hold to compare with original"

5. **Denoise / Re-process action button** (shown when NOT isProcessing):
   - If NOT hasDenoisedResult: "Denoise Audio" button (primary style) → viewModel.startDenoising(mergedFileURL: mergedFileURL)
   - If hasDenoisedResult: "Re-process" button (secondary style) → same action

6. **Export button** (shown when hasDenoisedResult):
   - Toolbar trailing button "Export"
   - Presents ExportFormatSheet on tap via `.sheet(isPresented: $showExportSheet)`
   - `onFormatSelected` handler MUST:
     1. Guard that `viewModel.denoisedTempURL` is non-nil (the intensity-blended .wav written by the ViewModel)
     2. Call `AudioMergerService.export(inputURL: viewModel.denoisedTempURL!, format: selectedFormat)`
     3. Present ExportProgressSheet by setting `showExportProgressSheet = true` and binding its `progress` to AudioMergerService's export progress
   - This exports the intensity-blended denoised audio (NOT the raw merged file passed into startDenoising)
   - Do NOT create a separate export path or new service — use the existing AudioMergerService.export API

7. **Progress modal** (non-dismissible sheet presented when isProcessing):
   - Reuse ExportProgressSheet: `ExportProgressSheet(progress: viewModel.progress, onCancel: { viewModel.cancelDenoising() })`
   - Present via `.sheet(isPresented: .constant(viewModel.isProcessing)) { ExportProgressSheet(...) }`
   - `.interactiveDismissDisabled(true)` to prevent swipe-down dismissal

8. **Error alert**: `.alert("Denoising Failed", isPresented: errorAlertBinding) { Button("OK") {} } message: { Text(viewModel.errorMessage ?? "") }`
